# Knowledge Base / RAG

Knowledge-base work belongs to the extension layer. It usually starts after the request side is already stable.

## When to read this

- you need retrieval-backed answers
- you want source-backed QA or document search
- you need KB results to feed a request or a workflow

## Where it fits

- stable request first
- retrieval and source injection second
- TriggerFlow only when retrieval becomes part of a larger workflow
